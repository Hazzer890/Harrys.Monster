Case study · Talk

Single-use software

Curtin Festival of AI Python LLM APIs Data analysis

The problem

Professionals meet problems every week that a small program would solve: reformat this data, cross-check these lists, chart this trend. Software was always too expensive to commission for one job, so people do these tasks by hand, or skip them.

The idea

AI collapses the cost of a small program to minutes. That makes a new category viable: single-use software. Build a deterministic tool for one job, verify it on cases you already know the answer to, run it, throw it away. You get the reliability of code, where the same input produces the same output every time, without the price tag that used to come with it. Asking a chatbot to do the task directly gives you neither the reliability nor the audit trail.

The talk

I presented this at Curtin's Festival of AI, with worked examples showing non-technical professionals how to get useful, trustworthy software out of AI tools: how to describe the job, how to verify the result, and when to trust it.

The practice

I use the method constantly. The most recent run: a series of single-use analysis tools for surfacing and visualising trends in moderate-sized datasets. Each took minutes to build and answered a question that would otherwise have stayed open.
