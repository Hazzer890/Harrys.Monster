How I work

Multi-model orchestration Agent workflows Daily practice

The claim

The sharpest evidence I can offer that I work at the current edge of AI practice is the environment I sit in every day.

The setup

Several models, each doing what it is best at. Agent workflows split tasks into roles. Routing rules send bulk mechanical work to cheap models and send taste-sensitive work, design, copy, API surfaces, to strong ones. Custom skill libraries encode my methods, so an agent picks up a task the way I would. Models review each other's output before I see it, and the loop catches mistakes I would have shipped.

Why this matters to a client

The same patterns transfer. Deciding which model does which job, encoding a team's standards so AI output actually meets them, and building review loops that catch errors before they reach a customer: that is AI implementation. I have already made the mistakes on my own projects, which is the cheapest place to make them.

Also in the toolkit

AI-generated technical documentation, kept accurate because it is regenerated from source rather than hand-maintained. And advising engineering students on applying AI to embedded-systems projects during my lab work at Curtin, which keeps me practised at explaining these systems to people meeting them for the first time.
