Case study

An agent that tunes machine-learning models

Python Claude API Agentic loop Curtin coursework

The problem

Hyperparameter tuning eats time. Train, wait, read the metrics, adjust, repeat. Each cycle needs judgement, but it is the same kind of judgement every time. That combination, a slow loop plus repeatable judgement inside known bounds, is exactly the shape of work an agent should take over.

The approach

I wrapped my training pipeline in an agentic loop built on the Claude API. The agent proposes a configuration, launches the run, reads the results, reasons about what to change, and tries again. It keeps a log of every attempt and the reasoning behind it. My role changed from turning knobs to setting the bounds, auditing the log, and stepping in when the reasoning drifted.

The outcome

Top marks on Machine Learning and Machine Perception assignments at Curtin, and a tuning process I supervised rather than babysat. The pattern generalises well past coursework: give an agent a measurable objective, a bounded search space, and an audit trail, and it will grind through iterations that would otherwise consume your evenings.
